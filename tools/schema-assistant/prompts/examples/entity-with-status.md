### Example: Entity with Status Workflow and Category

**Request**: Create a Service Request entity with a status workflow (New, Assigned, In Progress, Resolved, Closed), an assigned_to user field, a priority category (Critical, High, Medium, Low), and a description.

**Response**:

Service requests use the Status type for workflow tracking and Category for priority classification. The assigned_to field references civic_os_users for user display.

-- [DDL] Table creation
CREATE TABLE public.service_requests (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY NOT NULL,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ,
  display_name TEXT NOT NULL,
  description TEXT,
  status INT NOT NULL DEFAULT get_initial_status('service_request'),
  assigned_to UUID DEFAULT current_user_id(),
  priority INT,
  created_by UUID NOT NULL DEFAULT current_user_id()
);
